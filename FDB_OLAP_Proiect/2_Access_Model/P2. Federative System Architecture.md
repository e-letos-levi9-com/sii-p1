## IMPLEMENTARE ACCESS surse de date externe

Descrierea de mai jos prezinta modul de acces pentru fiecare sursa, folosind implementarea existenta in proiectul curent.

### DS_1: Oracle XE 21c (schema CREDITS_USER) - mecanism de access: oracle_fdw

Format/Tip de access:

- SQL (baza de date relationala Oracle)

Implementare acces la sursa de date externa in schema de integrare/mediere:

- descriere mecanism de access:
  - in PostgreSQL se foloseste extensia oracle_fdw
  - se defineste un server extern oracle_server catre Oracle containerizat
  - se defineste user mapping pentru utilizatorul postgres
  - se importa tabelele Oracle in schema de mediere oracle_movies prin IMPORT FOREIGN SCHEMA
- definitie structura de access: tabela/view remote sau locala:
  - remote foreign tables in schema oracle_movies (ex: movies, people, cast_members, crew_members)

Implementare comenzi SQL + alte_comenzi_setari_necesare:

- Script principal: for-oracle/postgres-init/01-install-oracle-fdw.sql

```sql
CREATE EXTENSION IF NOT EXISTS oracle_fdw;

CREATE SERVER oracle_server
  FOREIGN DATA WRAPPER oracle_fdw
  OPTIONS (dbserver '//oracle-credits-db:1521/XEPDB1');

CREATE USER MAPPING FOR postgres
  SERVER oracle_server
  OPTIONS (user 'credits_user', password 'credits_pass');

CREATE SCHEMA IF NOT EXISTS oracle_movies;
GRANT USAGE ON SCHEMA oracle_movies TO postgres;
```

- Script import schema Oracle: for-oracle/import-foreign-schemas.sql

```sql
IMPORT FOREIGN SCHEMA "CREDITS_USER" FROM SERVER oracle_server INTO oracle_movies;
```

- Setari infrastructura necesare (docker-compose):
  - host Oracle in retea: oracle-credits-db
  - port Oracle: 1521
  - service Oracle: XEPDB1

### DS_2: MongoDB 7 (database moviesdb) - mecanism de access: multicorn + wrapper mongo_fdw.py

Format/Tip de access:

- JSON (colectii MongoDB)
- SQL federat (prin foreign tables in PostgreSQL)

Implementare acces la sursa de date externa in schema de integrare/mediere:

- descriere mecanism de access:
  - in PostgreSQL se foloseste extensia multicorn
  - se configureaza serverul mongodb_server cu wrapper-ul Python mongo_fdw.MongoFDW
  - wrapper-ul custom este implementat in fisierul for-oracle/mongo_fdw.py
  - se creeaza foreign tables in schema mongo_movies pentru colectiile MongoDB
- definitie structura de access: tabela/view remote sau locala:
  - remote foreign tables in schema mongo_movies:
    - movies
    - genres
    - production_companies
    - production_countries

Implementare comenzi SQL + alte_comenzi_setari_necesare:

- Script principal: for-oracle/postgres-init/02-install-multicorn-mongo.sql

```sql
CREATE EXTENSION IF NOT EXISTS multicorn;

CREATE SERVER IF NOT EXISTS mongodb_server
FOREIGN DATA WRAPPER multicorn
OPTIONS (
    wrapper 'mongo_fdw.MongoFDW'
);

CREATE USER MAPPING IF NOT EXISTS FOR postgres
SERVER mongodb_server
OPTIONS (
    username 'admin',
    password 'admin123'
);

CREATE SCHEMA IF NOT EXISTS mongo_movies;
GRANT USAGE ON SCHEMA mongo_movies TO postgres;

CREATE FOREIGN TABLE IF NOT EXISTS mongo_movies.movies (
    _id TEXT,
    title TEXT,
    budget BIGINT,
    revenue BIGINT,
    runtime INTEGER,
    vote_average FLOAT,
    vote_count INTEGER,
    popularity FLOAT,
    release_date TEXT,
    overview TEXT,
    original_language TEXT,
    genres TEXT,
    production_companies TEXT,
    production_countries TEXT
)
SERVER mongodb_server
OPTIONS (
    host 'mongodb-movies',
    port '27017',
    database 'moviesdb',
    collection 'movies'
);
```

- Setari infrastructura necesare (docker-compose):
  - host MongoDB in retea: mongodb-movies
  - port MongoDB: 27017
  - database: moviesdb
  - credentials: admin / admin123

### DS_3: ratings.csv (fisier extern montat in container) - mecanism de access: file_fdw

Format/Tip de access:

- CSV (sursa primara)
- SQL (acces prin foreign table in PostgreSQL)

Implementare acces la sursa de date externa in schema de integrare/mediere:

- descriere mecanism de access:
  - in PostgreSQL se foloseste extensia file_fdw
  - se configureaza un server CSV (csv_server)
  - fisierul ratings.csv este expus ca foreign table csv_data.ratings
  - se defineste un view local csv_data.ratings_with_date pentru conversia timestamp
- definitie structura de access: tabela/view remote sau locala:
  - remote foreign table: csv_data.ratings
  - local view: csv_data.ratings_with_date

Implementare comenzi SQL + alte_comenzi_setari_necesare:

- Script principal: for-oracle/postgres-init/03-install-csv-fdw.sql

```sql
CREATE EXTENSION IF NOT EXISTS file_fdw;

CREATE SERVER IF NOT EXISTS csv_server
FOREIGN DATA WRAPPER file_fdw;

CREATE SCHEMA IF NOT EXISTS csv_data;
GRANT USAGE ON SCHEMA csv_data TO postgres;

CREATE FOREIGN TABLE IF NOT EXISTS csv_data.ratings (
    user_id INTEGER,
    movie_id INTEGER,
    rating NUMERIC(2,1),
    timestamp BIGINT
)
SERVER csv_server
OPTIONS (
    filename '/data/ratings.csv',
    format 'csv',
    header 'true',
    delimiter ','
);

CREATE OR REPLACE VIEW csv_data.ratings_with_date AS
SELECT
    user_id,
    movie_id,
    rating,
    timestamp,
    TO_TIMESTAMP(timestamp) AS rating_date
FROM csv_data.ratings;
```

- Setari infrastructura necesare (docker-compose):
  - montare volum in postgres: ../data:/data:ro
  - cale fisier in container: /data/ratings.csv
