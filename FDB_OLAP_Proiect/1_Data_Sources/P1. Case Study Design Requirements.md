## STABILIRE surse de date externe

Acest document descrie fiecare sursa externa folosita in proiectul de integrare federata Oracle + MongoDB + CSV.

### DS_1: Oracle XE (schema CREDITS_USER) - sursa externa pentru credite filme

Tip sursa de date:

- Tip model de date: relational
- Tip format de access: SQL (Oracle), cu incarcare initiala din CSV

Descriere structuri de date:

- campuri/coloane/atribute etc.:
  - movies(movie_id, created_at)
  - people(person_id, name, gender, profile_path)
  - cast_members(cast_id, movie_id, person_id, character_name, credit_id, cast_order)
  - crew_members(crew_id, movie_id, person_id, department, job, credit_id)
- legaturi intre structurile din cadrul aceleasi surse de date:
  - movies.movie_id -> cast_members.movie_id
  - movies.movie_id -> crew_members.movie_id
  - people.person_id -> cast_members.person_id
  - people.person_id -> crew_members.person_id

Implementare sursa de date externa:

- Script_Comenzi_DDL_DML (pentru format SQL/NoSQL):
  - for-oracle/init-scripts/01-create-schema.sql
  - for-oracle/import_data.py
- TextFile.csv: data/credits.csv

### DS_2: MongoDB 7 (database moviesdb) - sursa externa pentru metadate filme

Tip sursa de date:

- Tip model de date: ierarhic/document
- Tip format de access: JSON (documente MongoDB), cu incarcare initiala din CSV

Descriere structuri de date:

- campuri/coloane/atribute etc.:
  - movies(_id, id, title, original_title, original_language, overview, tagline, homepage, imdb_id, status, adult, video, budget, revenue, runtime, popularity, vote_average, vote_count, release_date, poster_path, genres, production_companies, production_countries, spoken_languages, belongs_to_collection, imported_at)
  - genres(_id, id, name)
  - production_companies(_id, id, name)
  - countries(_id, iso_3166_1, name)
- legaturi intre structurile din cadrul aceleasi surse de date:
  - movies.genres[*].id -> genres.id
  - movies.production_companies[*].id -> production_companies.id
  - movies.production_countries[*].iso_3166_1 -> countries.iso_3166_1

Implementare sursa de date externa:

- Script_Comenzi_DDL_DML (pentru format SQL/NoSQL):
  - for-mongo/init-scripts/01-create-user.js
  - for-mongo/import_movies.py
- Document.JSON: Colectiile movies, genres, production_companies, countries
- TextFile.csv: data/movies_metadata.csv

### DS_3: ratings.csv - sursa externa de rating-uri accesata federat

Tip sursa de date:

- Tip model de date: tabular (fisier CSV)
- Tip format de access: CSV direct + SQL prin foreign table

Descriere structuri de date:

- campuri/coloane/atribute etc.:
  - ratings(user_id, movie_id, rating, timestamp)
  - ratings_with_date(user_id, movie_id, rating, timestamp, rating_date)
- legaturi intre structurile din cadrul aceleasi surse de date:
  - ratings_with_date este view derivat din ratings
  - rating_date este calculat din timestamp prin TO_TIMESTAMP

Implementare sursa de date externa:

- Script_Comenzi_DDL_DML (pentru format SQL/NoSQL):
  - for-oracle/postgres-init/03-install-csv-fdw.sql
- TextFile.csv: data/ratings.csv

## Corelatii intre surse

- Corelatia principala intre DS_1, DS_2 si DS_3 este cheia movie_id (Oracle/CSV) respectiv _id/id (MongoDB).
- DS_1 contribuie cu date de cast/crew, DS_2 cu metadate de film, DS_3 cu rating-uri de utilizator.
- Interogarile federate folosesc conversii explicite de tip pentru join (ex. movie_id::text = _id).
