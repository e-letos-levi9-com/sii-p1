Documentul prezinta implementarea pe doua niveluri, conform modelului cerut: nivel de consolidare si schema analitica ROLAP.

(1) Nivel CONSOLIDARE date

View_Consolidare_1: integration_model.vw_movie_consolidated
Nume/descriere:
Consolidare la nivel de film intre metadate film (MongoDB), credite film (Oracle) si rating-uri utilizatori (CSV).
Surse de date integrate:
DS_1 + DS_2 + DS_3
Definitie:
CREATE OR REPLACE VIEW integration_model.vw_movie_consolidated AS ...

View_Consolidare_2: integration_model.vw_actor_movie_consolidated
Nume/descriere:
Consolidare la nivel actor-film, cu legatura intre actori/cast Oracle, metadate film MongoDB si rating-uri CSV.
Surse de date integrate:
DS_1 + DS_2 + DS_3
Definitie:
CREATE OR REPLACE VIEW integration_model.vw_actor_movie_consolidated AS ...

## Mapare explicita Query -> View (aliniere cu fisierele echipei)

1. Fisier queries_Diana.sql
- Query "RUNTIME IMPACT ON SUCCESS" -> integration_model.av_budget_runtime_rollup
- Query "RATING TRENDS OVER TIME" -> integration_model.av_language_decade_cube
- Query "GENRE PROFITABILITY ANALYSIS" reprezinta analiza exploratorie si este suportata de stratul fact + dimensiuni:
integration_model.fact_movie_performance + integration_model.dim_movie

2. Fisier queries_Eliza.sql
- Query "MOVIE SUCCESS METRICS" -> integration_model.av_budget_runtime_rollup
- Query "TOP PERFORMING ACTORS" -> integration_model.av_top_actors
- Query "DIRECTOR SUCCESS ANALYSIS" este suportat de stratul de consolidare actor/persoana:
integration_model.vw_actor_movie_consolidated + integration_model.dim_person

3. Fisier queries_Marius.sql
- Query "LANGUAGE/COUNTRY MARKET PERFORMANCE" -> integration_model.av_language_decade_cube
- Query "POPULARITY VS QUALITY" este suportat de stratul fact la nivel film:
integration_model.fact_movie_performance
- Query "RELEASE MONTH STRATEGY ANALYSIS" este suportat de dimensiunea de timp:
integration_model.dim_time_release + integration_model.fact_movie_performance

4. Rolul maparii
- fisierele queries_*.sql raman setul de interogari analitice ad-hoc pentru validare/explorare
- scriptul P3_integration_olap_views.sql ofera modelul stabil (view-uri reutilizabile) pentru rulare repetabila
- documentul P3 descrie formal aceasta corespondenta pentru evaluare
